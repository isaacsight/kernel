## Principles

Every feature we build follows these principles:

1. **Open by default** — everything is MIT, everything runs locally
2. **Science first** — if it helps a researcher, it's high priority
3. **Zero cost path** — every feature works with free/local models
4. **Community driven** — roadmap shaped by user feedback and RFCs
5. **Lean and fast** — 8 runtime dependencies, sub-second startup

## How to Influence the Roadmap

- **Vote**: React with a thumbs-up on GitHub issues
- **RFC**: Propose significant changes via GitHub Discussions
- **Build**: Submit a PR — the best way to get a feature shipped
- **Talk**: Join the Discord and share what you're working on
