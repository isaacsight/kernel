Project: META-03 | Self-Recursive Architecture

The Platform Itself

A recursive analysis of the static-dynamic hybrid architecture powering this research suite.

01. The Engine (Dynamic)

Under the hood, the platform runs specialized React applications for AI research. This is "The Other Computer"—the active, thinking part of the suite.

Observed Here: The Logit-Level Debugger (`apps/debugger`) intercepting a token stream to visualize probability distributions in real-time.

02. The Context (Visual)

We don't just process text; we see it as the model does.

Observed Here: The Context Window Visualizer (`apps/context-viewer`) rendering the sliding-window attention mechanism using D3.js and WebGPU.

03. The Gallery (Static)

The results of our research are curated in a high-performance static environment.

Observed Here: The Collections Interface, organizing scattered thoughts into coherent "Stacks" using a custom Python SSG.

04. The Interface Identity

The design language is strictly derived from the content it serves. Rather than an external UI kit, we built a system that reflects the dual nature of the platform: Research Data (Monospace) and Human Narrative (Sans-serif).

Typography as Interface

We use JetBrains Mono for headers, metadata, and code to evoke a terminal environment. Inter is reserved for the prose itself, ensuring long-form readability without breaking the "researcher" immersion.

The Terminal Grid

The homepage abandons the traditional blog roll for a High-Density Data Grid. Status indicators, date stamps, and tags are presented as raw data points, allowing the user to scan the corpus like a dataset.

05. The Complete Ecosystem

The architecture is designed for scale and versatility, extending beyond simple blog posts into a full-featured digital presence.

The Laboratory

A dedicated namespace for raw ideas, workshops, and technical POCs. It separates "finished work" from "thinking in public."

Universal Access

The entire platform is responsive, ensuring the terminal aesthetic translates perfectly to mobile viewports without losing data density.
