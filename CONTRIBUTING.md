# Contributing to Does This Feel Right?

First off, thank you for considering contributing to this project! It's people like you that make it an open source platform.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please search the [issues](https://github.com/isaachernandez/blog-design/issues) to see if it has already been reported. If not, open a new issue with a clear description of the problem and steps to reproduce it.

### Suggesting Enhancements
Have an idea for a new feature? We'd love to hear it! Open an issue to discuss your proposal.

### Pull Requests
1. Fork the repository.
2. Create a new branch (`git checkout -b feature/your-feature-name`).
3. Make your changes.
4. Run tests (`pytest`) to ensure everything is working correctly.
5. Commit your changes (`git commit -m 'Add some feature'`).
6. Push to the branch (`git push origin feature/your-feature-name`).
7. Open a Pull Request.

## Development Setup

See [DEVELOPMENT.md](DEVELOPMENT.md) for detailed instructions on setting up your local environment.

## Code Style

We use `black` for formatting and `ruff` for linting. Please ensure your code follows these standards before submitting a PR.

```bash
black .
ruff check . --fix
```
