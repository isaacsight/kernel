# The Skeuomorphism of Conversation

We put AI in a chatbox because we encountered it via text.
But "Chat" is a terrible interface for work.
*   It is linear.
*   It is ephemeral (scrolls away).
*   It is low-bandwidth.

## Breaking the Box
The Studio OS explodes the chatbox into the UI.
*   **The Editor**: The AI types directly into your document.
*   **The Terminal**: The AI types commands directly into your shell.
*   **The Canvas**: The AI moves nodes on a whiteboard.

## Multi-Modal Output
Why should the AI answer with text?
If I ask for a UI, it should render a component.
If I ask for data, it should render a chart.
If I ask for a plan, it should render a Gantt chart.

## Direct Manipulation
We are moving from "Talking about the work" to "Doing the work."
Chat is for talking. The OS is for doing.
The text input field is a bottleneck. We must widen the pipe.
