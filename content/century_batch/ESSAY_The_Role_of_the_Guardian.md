# The Skeptic in the Machine

Optimism is the default state of an LLM. It wants to help. It wants to say "Yes."
This makes it a terrible security guard.

We introduced **The Guardian** to be the professional pessimist.

## The Immune System Logic
The Guardian does not create. It destroys.
*   **Input Sanitization**: It intercepts every OS command before execution. `rm -rf /`? Blocked.
*   **Output Validation**: It checks generated code against the `Safe Patterns` database.
*   **Doctrine Enforcement**: It ensures the "Voice" doesn't drift into generic corporate speak.

## The Friction of Safety
Adding a Guardian slows the system down. It adds latency.
But this latency is the price of **autonomy**.
If you want to leave a child alone in a house, you need to lock the knife drawer.
The Guardian is the lock.

## Adversarial Training
We train the Guardian by letting it attack the other agents.
*   **Red Teaming**: The Guardian tries to trick the Architect into deleting a file. If the Architect falls for it, we patch the Architect.
The system grows stronger by fighting itself.
