# Static is Dead

The era of the "Brochure Website" is over.
A website that doesn't change when you look at it is a PDF.
We are building **Living Webs**.

## Metabolism
A living thing takes in energy and puts out waste.
*   **Input**: The Studio OS consumes RSS feeds, user clicks, and git commits.
*   **Metabolism**: The **Council** processes this data, digesting it into insights.
*   **Output**: The site updates its content, its layout, and its priorities.

## Growth Rings
Most sites grow by appending blog posts.
A Living Web grows by **Evolution**.
The CSS itself should change. If users read more at night, the site should drift towards dark mode by default. If users prefer code snippets, the font size of `<pre>` tags should increase.
The site adapts to its environment.

## The Gardener
The role of the webmaster shifts from "Builder" to "Gardener."
You don't place every brick. You water the soil (feed data) and prune the branches (curate output).
The site grows itself.
