# Naming the Ghost

A shell script is `deploy.sh`.
An Agent is "The Architect."
Why do we do this?

## The Social Interface
We use anthropomorphism as a **Compression Algorithm**.
It is easier to predict "How Arthur would react" than to predict "How a transformer model with temperature 0.7 would react."
If we model the AI as a person, our social brain (which is huge) kicks in.

## Empathy for the Machine
We found that when we gave the agents names and avatars, we treated them better.
We wrote clearer prompts. We said "Please."
And strangely, the results got better.
The "Politeness Quality Improvement" is a real phenomenon in LLMs.
Treating the machine like a person makes it a better machine.

## The Hazard
The danger is forgetting it is a machine.
We must maintain the "Ironic Distance."
It is *like* a person. It is *not* a person.
We name the script to make it usable, not to make it friend.
